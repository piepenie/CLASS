1. 명령어의 구조
명령어는 연산코드와 오퍼랜드로 나뉨

연산코드 : 수행할 연산

오퍼랜드 : 연산에 사용될 데이터 or 
연산에 사용될 데이터가 저장된 위치

★기계어, 어셈블리어도 이 명령어에 포함되있음.

2. 오퍼랜드
연산에 사용될 데이터가 저장된 위치 : 주소 필드

오퍼랜드가 없는 경우도 있고 한 개 이상인 경우도 있다.

오퍼랜드가 없는 경우 : 0 주소 명령어
오퍼랜드가 1개 : 1주소 명령어
오퍼랜드가 2개 : 2주소 명령어
오퍼랜드가 3개 : 3주소 명령어 

라고함. 

3. 연산코드

데이터 전송 
산술/논리 연산
제어 흐름 변경
입출력 제어 

3-1. 데이터 전송
MOVE : 데이터를 옮겨라
STORE : 메모리에 저장하라
LOAD(FETCH)  : 메모리에서 CPU로 데이터를 가져와라
PUSH : 스택에 데이터를 저장하라
POP : 스택의 최상단 데이터를 가져와라 
<< 이건 암기가 아니라 유형이 익숙해지게 눈도장 찍어놓기. 

★스택(STACK) << 매우 중요

라스트 인 퍼스트 아웃 : 가장 처음들어간게 가장 마지막에 나오는 과정
(LIFO)

★큐(QUEUE) << 이것도 매우 중요
양쪽 끝이 뚫려 있는 구조
퍼스트 인 퍼스트 아웃 : 가장 처음들어간게 가장 먼저 나오는 과정
(FIFO)

3-2. 산술/논리 연산
ADD / SUBTRACT / MULTIPLY / DIVIDE : 사칙연산 (덧,뺼,곱,나눗셈) 수행해라
INCREMENT / DECREMENT : 오퍼랜드에 1을 더해라, 1을 빼라
AND/OR/NOT : AND / OR / NOT 연산을 수행해라
COMPARE : 두 개의 숫자 또는 TRUE / FALSE 값을 비교하라 


3-3. 제어 흐름 변경
JUMP : 특정 주소로 실행 순서를 옮겨라 
CONDITIONAL JUMP : 조건에 부합할 때특정 주소로 실행 순서를 옮겨라
HALT : 프로그램의 실행을 멈춰라
CALL : 되돌아올 주소를 저장한 채 특정 주소로 실행 순서를 옮겨라
RETURN : CALL 을 호출할 때 저장했던 주소로 돌아가라 

3-4. 입출력 제어 
READ(INPUT) : 특정 입출력 장치로부터 
WRITE(OUTPUT) : 특정 입출력 장치로 데이터를 써라
START IO : 입출력 장치를 시작하라
TEST IO : 입출력 장치의 상태를 확인하라

TIP: 오퍼랜드에 값을 저장하지 않고 주로 주소값을 저장하는 이유는
그 오퍼랜드에 표현할 수 있는 데이터 크기가 작기때문!

4. 명령어 주소 지정 방식

4-1. 유효 주소(effective address)
연산에 사용할 데이터가 저장된 위치 
 
4-2. 명령어 주소 지정 방식(addressing modes)

연산에 사용할 데이터가 저장된 위치를 찾는 방법
유효 주소를 찾는 방법
다양한 명령어 주소 지정 방식들

①즉시 주소 지정 방식(immediate addressing mode)
연산에 사용할 데이터를 오퍼랜드 필드에 직접 명시 
가장 간단한 형태의 주소 지정 방식
연산에 사용할 데이터의 크기가 작아질 수 있지만, 빠름

②직접 주소 지정 방식(direct addresing mode)
오퍼랜드 필드에 유효 주소 직접적으로 명시
유효 주소를 표현할 수 있는 크기가 연산 코드만큼 줄어듦

③간접 주소 지정 방식(indirect addresing mode)
오퍼랜드 필드에 유효 주소의 주소를 명시
앞선 주소 지정 방식들에 비해 속도가 느림
대신 충분하게 유효주소를 크게 가져올 수 있음.

④레지스터 주소 지정 방식(register addresing mode)
연산에 사용할 데이터가 저장된 레지스터 명시
★메모리에 접근하는 속도보다 레지스터에 접근하는 것이 빠름
(매우 중요)

⑤레지스터 간접 주소 지정 방식(register indirect addressing mode)
연산에 사용할 데이터를 메모리에 저장
그 주소를 저장한 레지스터를 오퍼랜드 필드에 명시
