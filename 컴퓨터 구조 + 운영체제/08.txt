1. 레지스터 

CPU 내부에는 다양한 레지스터들이 있고, 각기 다른 역할을 한다

2. 반드시 알아야하는 레지스터 

① 프로그램 카운터 : 메모리에서 가져올 명령어의 주소(메모리에서 읽어 들일 명령어의 주소)

② 명령어 레지스터 : 해석할 명령어(방금 메모리에서 읽어 들인 명령어)

③ 메모리 주소 레지스터 : 메모리의 주소

④ 메모리 버퍼 레지스터 : 메모리와 주고받을 값(데이터와 명령어)

⑤ 플래그 레지스터 : 연산 결과 또는 CPU 상태에 대한 부가적인 정보

⑥ 범용 레지스터 : 다양하고 일반적인 상황에서 자유롭게 사용

7~8번은 특정 레지스터를 이용한 주소 지정 방식에서 사용함.
⑦ 스택 포인터 
⑧ 베이스 레지스터 

3. 특정 레지스터를 이용한 주소 지정 방식(1): 스택 주소 지정 방식
스택 주소 지정 방식 : 스택과 스택 포인터를 이용한 주소 지정 방식

스택 포인터 : 스택의 꼭대기를 가리키는 레지스터(스택이 어디까지 
차 있는지에 대한 표시)

ex) 스택 포인터가 4번지를 가르키고 있다면, 현재 4번지까지 채워져있는거임. 약간 물높이를 알려주는 느낌.

+
메모리 안에 스택을 사용할 수 있는 구역이있음 이를 스택 영역 이라고함

4. 특정 레지스터를 이용한 주소 지정 방식(2): 변위 주소 지정 방식
변위 주소 지정 방식 : 오퍼랜드 필드의 값과 특정 레지스터의 값을 더하여
유효 주소 얻기
>> 여기서도 두가지가 있음

특정 레지스터(프로그램 카운터 or 베이스 레지스터) 두 가지 중 하나의 
값을 오퍼랜드 필드의 값(변위)와 더하는 방식임.

4-1. 상대 주소 지정 방식
PC(프로그램 카운터) + 오퍼랜드 값을 더하여 유효 주소를 얻는 방식
PC에 읽어들일 명령어 주소가 저장되어 있을때, 오퍼랜드 값이 만약
-3 이면 3번째 이전 번지로 찾아가 그 명령어를 실행하는 것
+3 이면 3번째 이후 명령어를 실행함.

4-2. 베이스 레지스터 주소 지정 방식
베이스 레지스터 + 오퍼랜드 값을 더하여 유효 주소를 얻는 방식
가령 베이스 레지스터 값이 200이고 오퍼랜드 값이 50이면 
250 번지를 실행하라는 뜻.

* 유효 주소 : 연산할 데이터가 있는 위치.
* 명령어 주소 : 실행할 명령어가 있는 위치
